selectedActivity = ddActivity and ddActivity:GetValue()

**Step 1 — Extend TI.Reset():**

At the end of the existing `TI.Reset()` function body (after the timezone reset block ending
with `if ddTz then ddTz:SetValue(selectedTzId) end`), add:

```lua
-- Activity reset (per D-07)
if ddActivity      then ddActivity:ClearValue()      end
if ddExactActivity then ddExactActivity:ClearValue()  end
if lblExactActivity then lblExactActivity:Hide()      end
if ddExactActivity  then ddExactActivity:Hide()       end
selectedActivity = nil
```

Per D-07: Both dropdowns cleared, exact row hidden, upvalue reset to nil.
Per Pitfall 5: `selectedActivity = nil` is critical — ClearValue() only clears the widget's
internal state, not the module-scope upvalue. Without this, the next Save would pass the
old activity through the guard.

**Step 2 — Extend TI.Populate():**

At the end of the existing `TI.Populate()` function body (after the date/time restoration block),
add:

```lua
-- Activity restore (per D-08)
if ddActivity and my.primaryActivity then
    ddActivity:SetValue(my.primaryActivity)
    selectedActivity = my.primaryActivity
    local subs = OW.ACTIVITY_SUBS and OW.ACTIVITY_SUBS[my.primaryActivity] or {}
    if #subs > 0 then
        ddExactActivity:SetItems(exactItemsForActivity(my.primaryActivity))
        if my.exactActivity then ddExactActivity:SetValue(my.exactActivity) end
        if lblExactActivity then lblExactActivity:Show() end
        ddExactActivity:Show()
    else
        if lblExactActivity then lblExactActivity:Hide() end
        ddExactActivity:Hide()
    end
end
```

Per D-08: Restores primaryActivity and exactActivity from myEntry, then triggers the same
show/hide logic as the primary dropdown's onChange handler. If the restored activity has
sub-types (e.g., Raid), the exact row is shown and the exact dropdown is populated and set.
If it has no sub-types (e.g., Quest), the exact row stays hidden.

Per Pitfall 4: The show/hide logic MUST be replicated here — just calling SetValue is not enough
because SetValue doesn't trigger the onChange handler.

IMPORTANT: `my` refers to the return value of `OnlineWhen.GetMyEntry()` which is already
declared at the top of TI.Populate(). The `my.primaryActivity` and `my.exactActivity` fields
were added by Phase 2 (SaveMyEntry stores them in OnlineWhenDB.myEntry).

IMPORTANT: `exactItemsForActivity` is the builder function added in Plan 01, Task 2. It must
be in scope (it's a local function defined earlier in the file).
