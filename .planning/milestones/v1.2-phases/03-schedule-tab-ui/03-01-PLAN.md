**Step 2 — Add item builder functions** (after `tzItems()` function, around line 220):

```lua
local function activityItems()
    local items = {}
    for _, act in ipairs(OW.ACTIVITY_LIST) do
        items[#items+1] = { value = act.label, label = act.label }
    end
    return items
end

local function exactItemsForActivity(activityLabel)
    local subs = OW.ACTIVITY_SUBS and OW.ACTIVITY_SUBS[activityLabel] or {}
    local items = {}
    for _, sub in ipairs(subs) do
        items[#items+1] = { value = sub, label = sub }
    end
    return items
end
```

Note: `value = act.label` (not `act.id`) because OW.ACTIVITY_SUBS is keyed by label string
and OW.SaveMyEntry stores primaryActivity as the label. Per research anti-pattern guidance.

**Step 3 — Build Group 3 in TI.Build()** (after the Save button block, before the closing `end` of TI.Build):

Actually, insert the Group 3 block BEFORE the Save button creation (before line 474 `saveBtn = CreateFrame...`).
The Save button is anchored to BOTTOM so order doesn't matter functionally, but logically
the group should appear before the button in code.

```lua
-- ============================================================
-- GROUP 3 — Activity
-- ============================================================
local activityGroupHeight = 152
local activityGroupY = dateTimeGroupY - dateTimeGroupHeight - 8

local activityGroup = makeGroupBox(parent, "Activity", groupMargin, activityGroupY, groupWidth, activityGroupHeight)

curY = CONTENT_Y

-- Primary activity row
lbl(activityGroup, "Activity", contentLeft, curY)
curY = curY - 20

ddActivity = makeDropdown(activityGroup, "OWDdActivity", activityItems(), nil,
    contentLeft, curY, 200,
    function(v)
        selectedActivity = v
        local subs = OW.ACTIVITY_SUBS and OW.ACTIVITY_SUBS[v] or {}
        if #subs > 0 then
            ddExactActivity:ClearValue()
            ddExactActivity:SetItems(exactItemsForActivity(v))
            lblExactActivity:Show()
            ddExactActivity:Show()
        else
            ddExactActivity:ClearValue()
            lblExactActivity:Hide()
            ddExactActivity:Hide()
        end
    end,
    "— Select —")
curY = curY - 36

-- Exact activity row (hidden by default per D-03)
lblExactActivity = lbl(activityGroup, "Specific Activity", contentLeft, curY)
curY = curY - 20

ddExactActivity = makeDropdown(activityGroup, "OWDdExactActivity", {}, nil,
    contentLeft, curY, 200, nil, "— Select —")

-- Hide exact row on creation (Pitfall 3: widgets are visible by default)
lblExactActivity:Hide()
ddExactActivity:Hide()
```

IMPORTANT per Pitfall 1: In the onChange handler, call `ddExactActivity:ClearValue()` BEFORE
`ddExactActivity:SetItems()`. This prevents the SetItems clamp behavior from auto-selecting
the last item of the new list when switching between activities that both have sub-types.

Per D-01: The group box height is fixed at 152px regardless of whether exact row is shown/hidden.

Per D-03: Exact row is hidden (not disabled) for Quest/Farm/Chill.
